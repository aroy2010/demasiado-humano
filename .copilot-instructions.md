# Instrucciones de Contexto para el Proyecto "AROY1962"

## Perfil del Proyecto
- **Tecnología:** Hugo (Generador de sitios estáticos).
- **Tema:** Blowfish.
- **Estructura de Configuración:** 100% Modular en la carpeta `config/_default/`. NOTA: No existe el archivo `hugo.toml` en la raíz.
- **Arquitectura de Contenido:** Leaf Bundles (`content/seccion/año/mes/slug/index.md`).
- **Despliegue:** GitHub conectado a Cloudflare Pages.

## Estructura Completa del Menú (menus.en.toml)
Copilot debe respetar estrictamente esta estructura de secciones y rutas:
- **Principio:** Apunta a la raíz (`/`) y es la Home Page.
- **Reflexiones:** Apunta a la sección `/posts`.
- **Ensayos:** Apunta a la sección `/escritos_creativos`.
- **Comentando:** Apunta a la sección `/lecturas`.
- **Reseñas:** Apunta a la sección `/resena`.
- **Acerca de:** Apunta a la sección `/acerca`.

## Directrices de Diseño y Contenido
1. **Imágenes Locales:** Cada publicación es un Leaf Bundle. Las imágenes (incluyendo la imagen destacada `feature.jpg`) deben vivir en la misma carpeta que el archivo `index.md`.
2. **Estética Visual:** Se busca un diseño atractivo, de alta legibilidad para textos largos, con un ancho de contenedor óptimo (estilo `max-w-prose` / 75-80 caracteres por línea) y tipografía Serif cómoda para lectura prolongada.

## Reglas para Copilot
- Antes de proponer código de diseño, verifica que sea compatible con las clases de Tailwind CSS que utiliza el tema Blowfish.
- Al sugerir cambios en la configuración, edita directamente los archivos en `config/_default/` (como `params.en.toml` o `menus.en.toml`). Nunca sugieras crear un archivo `hugo.toml`.
